## 一、项目概述

### 1.1 项目背景

企业内线电话业务管理平台（PhoneBiz），用于统一管理企业内部电话资源的全生命周期，包括号码目录管理、工单流转、账单分摊与发票分发。目标用户为行政、IT运维、财务和高层管理人员，**不面向普通员工**。

### 1.2 组织架构模型

- 多层级集团组织架构（集团 → 一级子公司 → 二级子公司/部门 → 无限层级）
- 人员角色：Admin（行政）、IT/Ops（运维）、Finance（财务）、Boss（老板）
- Admin 管理范围按组织层级定义，支持按组织结构树灵活分配
- 普通员工无系统访问权限

### 1.3 系统边界

- **不含**：OA 集成（不依赖现有 OA 系统）、移动端 App
- **不含**：与财务系统的自动对接（发票分发和对账为人工线下补充）
- **包含**：Excel 账单导入、PDF 发票上传与分发

### 1.4 技术选型

| 层级 | 技术 |
|------|------|
| 前端 | React |
| 后端 | Spring Boot |
| 数据库 | MySQL 8 |
| 部署 | 待定 |

## 二、功能模块

### 模块一：组织架构管理（基础模块）

#### 2.1.1 集团组织结构树

- 支持多层级：集团 → 子公司 → 部门 → 子部门（无限深度）
- 树形结构展示，支持增删改查
- 部门属性：名称、所属上级、类型（集团总部部门 / 子公司 / 其他）
- 子公司属性：名称、一级/二级标识

#### 2.1.2 员工管理

- 员工字段：工号（6位数字或字母）、姓名、部门、职位、联系方式
- 工号规则：6位字符；纯数字则直接作为分机号（去掉前导0，分机号不以0开头）；含字母则由系统随机分配6位数字分机号
- 每个部门分配独立分机号池（由系统管理员配置）
- 员工离职：管理员提交工单 → 运维操作 → 号码回收至闲置池

#### 2.1.3 成本中心对照表

- 字段：部门、成本中心名称、成本中心代码
- 维护权限：集团总部财务（仅维护集团总部各部门）
- 用途：账单分配时将费用归属到对应成本中心
- 同一部门可对应多个成本中心（多对多关系）

### 模块二：电话目录管理（核心模块）

#### 2.2.1 号码生命周期

| 状态 | 含义 | 说明 |
|------|------|------|
| 空闲（idle） | 未分配 | 可直接分配 |
| 在用（active） | 正常使用 | 已分配给员工 |
| 停机（stopped） | 暂停使用 | 可恢复 |
| 已拆机（cancelled） | 已销号 | 归还运营商，不可恢复 |

**状态转移规则：**
- 空闲 → 在用：分配号码给员工（工单 allocate）
- 在用 → 空闲：员工离职，号码回收（工单 reclaim）
- 在用 → 停机：欠费/申请暂停（工单 trouble）
- 停机 → 在用：恢复使用（工单 reclaim/trouble）
- 在用/停机 → 已拆机：销号归还运营商（工单 surrender）
- 空闲 → 已拆机：直接销号（工单 surrender）

#### 2.2.2 号码属性

| 字段 | 说明 |
|------|------|
| phone_number | 电话号码（全局唯一） |
| user_id | 归属员工工号（空闲时为空） |
| extension_number | 分机号（全局唯一） |
| status | 状态（idle/active/stopped/cancelled） |
| is_shared | 是否共享号码（boolean） |
| org_id | 归属部门 ID |
| created_at / updated_at | 时间戳 |

#### 2.2.3 号码操作（工单触发）

从号码列表页可直接发起以下操作：

| 操作 | 类型 | 说明 |
|------|------|------|
| allocate | 分配 | 将空闲号码分配给员工 |
| change_user | 过户 | 变更号码使用人 |
| change_number | 换号 | 更换号码 |
| change_org | 转移 | 变更归属部门 |
| reclaim | 回收 | 员工离职，号码回池 |
| surrender | 拆机 | 号码归还运营商 |
| trouble | 停复机 | 申请停机/复机 |

#### 2.2.4 分机号池管理

- 系统管理员按部门分配分机号池范围
- 部门管理员只能分配本部门池内分机号
- 分机号全局唯一，不可重复

#### 2.2.5 历史与快照

**phone_history（永久留存，无保留期限）：**
- 记录号码所有状态变更、操作人、操作时间
- 不可删除，用于审计追踪

**phone_surrender_record（永久归档）：**
- 记录所有已拆机号码的最终信息
- 已拆机号码不参与后续分摊

**phone_snapshot（月度快照）：**
- 每月生成一次快照，匹配账单分摊
- 包含字段：phone_number、user_id、extension_number、status、org_id、成本中心代码、月份
- 快照纳入：active + stopped（可分配）；idle 不纳入；已拆机纳入但标记为"已拆机/不参与分摊"
- 快照数据锁定后不允许修改

### 模块三：工单系统

#### 2.3.1 工单类型

| 类型 | 触发动作 | 说明 |
|------|----------|------|
| allocate | 分配号码 | 将空闲号码分配给员工 |
| change_user | 过户 | 变更号码使用人 |
| change_number | 换号 | 更换号码 |
| change_org | 转移 | 变更号码归属部门 |
| reclaim | 回收 | 员工离职号码回池 |
| surrender | 拆机 | 号码归还运营商 |
| trouble | 停复机 | 申请停机或复机 |

#### 2.3.2 工单流程

```
[行政/管理员创建] → [运维接收] → [运维处理] → [运维完成] → [管理员归档]
```

- 无审批节点
- 无超时规则
- 无邮件通知，仅系统消息推送
- 支持批量工单（跨组织自动拆单）

#### 2.3.3 批量工单

- 管理员可一次性提交多个号码的工单
- 如涉及多个部门，系统自动按部门拆分为独立工单
- 工单处理结果分别记录

#### 2.3.4 工单通知

- 仅系统消息推送（无需邮件/短信）
- 推送对象：相关管理员、运维、财务（如涉及账单）

### 模块四：账单管理

#### 2.4.1 账单导入

- 格式：Excel（.xlsx）
- 操作角色：运维（Ops）
- 导入时需选择对应账单月份
- 账单列与系统字段的映射关系：**待确认（Excel列名未定）**
- 导入后数据进入账单原始表（raw），等待后续处理

#### 2.4.2 账单分摊流程

```
[运维导入账单] → [系统匹配快照组织归属] → [生成待确认分摊明细] → [行政确认部门归属正确性] → [行政确认费用金额正确性] → [财务确认异常标记] → [财务确认提交]
```

| 步骤 | 操作角色 | 确认内容 |
|------|----------|----------|
| 1 | 运维 | 导入 Excel，更新账单月 |
| 2 | 系统 | 自动匹配 snapshot 归属 |
| 3 | 行政 | 确认部门归属正确 |
| 4 | 行政 | 确认金额正确 |
| 5 | 财务 | 确认异常标记 |
| 6 | 财务 | 提交确认 |

- 系统对分摊数据进行异常检测（金额异常/与历史差异大），自动标记待审
- 所有账单数据永久保留，无删除权限

#### 2.4.3 子公司对账

- 子公司财务查看旗下各部门分摊汇总数据
- 汇总后向集团财务发起对账确认
- 确认方式：**待补充**（线上确认 or 线下补充系统）

### 模块五：发票分发（新增模块 v1.1）

#### 2.5.1 概述

集团总部统一收集并上传各子公司发票 PDF，系统自动识别发票中的公司名称，匹配对应子公司财务账户，自动分发。

#### 2.5.2 发票格式

- 格式：PDF（每个文件独立）
- 命名约定：`{公司名称}_序号.PDF`（序号为纯流水号，无固定位数要求）
- 公司名称出现在 PDF 文件内容中（通过 OCR/文字提取识别）

#### 2.5.3 发票导入流程

```
[集团总部财务上传 PDF（批量）] → [选择账单月份] → [系统自动提取PDF内公司名称]
→ [匹配子公司财务账户] → [自动分发至对应子公司财务]
```

#### 2.5.4 发票分发规则

| 发票来源 | 分发目标 | 识别方式 |
|----------|----------|----------|
| 集团总部发票 | 集团总部财务 | 公司名称="集团总部" |
| 各一级子公司发票 | 对应子公司财务 | 从 PDF 内容提取公司名称，匹配组织架构 |

- 匹配逻辑：PDF 内容提取文本 → 匹配组织架构中的公司名称
- 无法识别的发票：标记为"待人工确认"，通知集团总部财务处理
- 已分发发票状态：待确认 → 子公司财务已读 → 财务确认

#### 2.5.5 发票状态

| 状态 | 含义 |
|------|------|
| 待识别 | 上传后尚未处理 |
| 待分发 | 已识别，等待分发 |
| 已分发 | 已推送给对应子公司财务 |
| 待确认 | 子公司财务尚未确认 |
| 已确认 | 子公司财务已确认 |
