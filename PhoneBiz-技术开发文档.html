1系统概述

PhoneBiz 是一个企业内线电话管理平台，面向集团型企业提供电话号码全生命周期管理、话机设备管理、工单流程驱动、账单分摊及发票处理等核心能力。采用模块化单体架构，27个模块按数据依赖自底向上开发。

核心能力
  • 号码全生命周期管理（6状态流转：idle→active→stopped→cancelled/reserved/disabled）
  • 话机设备管理与SIP终端注册
  • 工单驱动的号码操作流程
  • 账单导入、分摊、异常检测
  • 发票OCR识别与自动分发对账
  • 多维度报表与Excel导出
设计原则
  • 金融级安全管控（JWT鉴权 / BCrypt加密 / 审计日志）
  • 模块化单体架构（27模块按数据依赖自底向上）
  • 契约驱动开发（DTO先行、后端先行、同PR提交）
  • 悲观锁并发控制（号码分配/换号双锁防死锁）
  • 功能开关灰度发布（工单驱动API迁移）
用户角色
Admin系统管理员，管理本部门及下级组织
Ops运维人员，号码/话机/号池日常操作
Finance财务人员，成本中心/账单/发票/对账
Boss管理层，全系统只读查看

2技术架构

2.1 整体架构

浏览器 (React 18 SPA)
↓ HTTP/REST
Nginx 反向代理 (CSP / CORS)
↓ /phonebiz/ → :8080/api/
Spring Boot 3.2 (JPA / JWT / Flyway)
↓ JDBC
MySQL 8.0 (InnoDB)

架构选型：采用模块化单体（Modular Monolith），模块间通过包隔离共享同一数据库和部署单元。27个模块按数据依赖关系自底向上开发，每个模块独立交付可验收。

2.2 后端架构（分层结构）

com.phonebiz ├── PhoneBizApplication.java // 启动类 ├── common/ // 通用框架层 │ ├── ApiResponse.java // 统一响应 (code, message, data) │ ├── BusinessException.java // 业务异常 (ErrorCode) │ ├── ErrorCode.java // 错误码枚举（按模块号段） │ └── GlobalExceptionHandler.java // @RestControllerAdvice ├── config/ // 配置层 │ ├── SecurityConfig.java // Spring Security + JWT │ ├── CorsConfig.java // 跨域配置 │ └── AsyncConfig.java // 异步线程池 ├── security/ // 安全层 │ ├── JwtUtil.java // JWT (HMAC-SHA256) │ ├── JwtAuthenticationFilter.java // JWT过滤器 │ └── UserDetailsServiceImpl.java // 用户认证 ├── entity/ (24个) // JPA实体 ├── repository/ (29个) // 数据访问 ├── service/ (26个) // 业务逻辑 ├── controller/ (22个) // 控制器 └── dto/ (34个) // 数据传输对象

2.3 前端架构

frontend/src/ ├── main.tsx // 入口 (React 18 createRoot) ├── App.tsx // 路由 + 布局 + 菜单 ├── index.css // 全局样式 ├── stores/authStore.ts // Zustand 认证状态 ├── api/request.ts // axios实例 + 拦截器 ├── api/modules/ (15个) // API模块 ├── pages/ (16个) // 页面组件 ├── components/ // 公共组件 └── types/ (11个) // TypeScript类型

2.4 部署架构

服务器：openEuler 7.0.5 (aarch64)，OrbStack VM，192.168.139.161
Nginx：反向代理 /phonebiz/ → http://127.0.0.1:8080/api/，含CSP安全头
后端：gradle bootRun --no-daemon，端口 8080，context-path /api
前端：Vite构建 → dist/ → Nginx静态服务
数据库：MySQL 8.0，phonebiz库

4数据库设计

4.1 ER关系总览

核心实体关系：
org_structure ← 1:N → employee / cost_center_mapping / extension_pool / phone_device / area_code_org_mapping
employee ← 1:1 → sys_user（通过employee_no关联）
phone_number ← 1:N → phone_history / phone_surrender_record
phone_device ← M:N → phone_number（通过 device_phone_mapping）
work_order ← 1:N → work_order_item
bill_raw ← 1:N → bill_allocation
invoice ← 1:N → invoice_file / invoice_distribution
phone_snapshot ← 月度快照，关联号码+组织+成本中心

7安全设计

7.1 认证与授权

措施实现
认证方式JWT（HMAC-SHA256 + jti唯一ID），30分钟过期
密码存储BCrypt哈希（cost=12），JWT密钥环境变量
角色体系4角色：admin/ops/finance/boss
权限控制@PreAuthorize按角色 + scope_org_id按组织裁剪
登录锁定5次失败→锁定30分钟→Admin解锁
首次登录forceChangePassword标志，强制修改密码
传输安全生产环境强制HTTPS

7.2 数据安全分级

级别内容访问
公开组织名称、部门列表登录用户
内部员工姓名、工号、分机号scope授权
机密电话号码、账单金额角色最小权限
绝密密码、JWT密钥认证模块内部

7.3 审计日志

登录/登出（用户名+IP+时间+结果）、权限变更、拆机确认、账单提交、发票分发。号码操作记录在 phone_history，话机操作记录在 phone_device_history。日志中电话号码脱敏为 138****1234 格式。

11开发流程规范

11.1 6步开发流程

Step 1 合同 → Step 2 数据库 → Step 3 后端 → Step 4 契约验证 → Step 5 前端 → Step 6 门禁 ↑ ↑ ↑ 定义DTO+端点列表 curl验证所有端点 对接已验证API (后端/前端类型对齐) ← 后端跑通前不写前端 → ← 不盲调 →
步骤内容验证方式
S1 合同定义Java DTO + TS类型 + API端点列表字段名/类型/枚举前后端一致
S2 数据库Flyway迁移文件MySQL执行成功
S3 后端Entity→Repository→Service→ControllerbootRun + curl每个端点
S4 契约验证curl验证所有端点响应符合DTO后端跑通前不写前端
S5 前端页面→组件→对接已验证API手动操作流程完整
S6 门禁G1-G5全部通过阻断项全部通过

11.2 端点粒度循环策略

简单CRUD模块(≤5端点)：整模块线性执行。
复杂业务模块(>5端点或含并发)：Step3-5循环，每完成1个端点的后端→curl→前端，再开始下一个。端点锁定后不再改动。
铁律：后端跑通前禁止写任何前端代码。
